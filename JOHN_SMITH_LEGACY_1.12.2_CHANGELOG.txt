====================================================================================================

John Smith Legacy - Resource Pack Changelog

for Vanilla Minecraft 1.12

http://www.johnsmithlegacy.co.uk/

NOTE:- This Resource Pack has been tested with Optifine, it's not required but does add some nice
       features like Connected Textures (CTM), Better Skies, Random Mobs and Custom Colours.

       Tested with the litemod VoxelMap by MamiyaOtaru.

       It's not been tested with any other mod and I cannot help or support other untested mods.

       This Resource Pack is not compatible with releases older than Minecraft 1.12

====================================================================================================

-{v1.4.0}- 29th February 2016 - Minecraft 1.9.0-----------------------------------------------------

Here we go; Minecraft 1.9.0 is out and is now the current release in the launcher.

Over the last month or so I've been going through the pack, cleaning up, making lots of consistency
and quality improvements.

I decided to add some custom models without them standing out too much and being too obvious in the
game, things like the Ladder now has some depth, Trapdoors and Doors now have edges too, more to
come in the future.

Some hand held items were rendered using flat 2D sprites but have internal 3D models when placed on
the ground, so why not use the model that already exists in the game, so items like the Brewing
Stand, Cauldron, Comparator, Flower Pot, Hopper, Repeater and Tripwire Hook now all use the 3D
models when held in the hand, on your head, on the ground or in your inventory.

There've been hundreds of changes to the pack to make it compatible with 1.9.0 and lots of tweaks
and additions, see the full changelog below for all the details and go play some Minecraft.

Tweaked all the fonts, also finally added the very hard to edit glyph_sizes.bin, this file is used
by Minecraft for the unicode character widths (kerning).

Note that the glyph_sizes.bin in only loaded when Minecraft loads, so you have to load Minecraft,
select the Resource Pack, exit Minecraft and then reload Minecraft, you only have to do this if
you need to use the Force Unicode Font option, this is a work around to a known Minecraft bug but at
least it works for now.

https://bugs.mojang.com/browse/MC-41270

The unicode_page_00 and unicode_page_01 fonts cover most of the diacritic marks used in the 
language regions like Germanic, Celtic, Romance, Slavic, Baltic, Finno-Ugric, Turkic and others.

Hopefully this will be good news to all the non English speaking players that use the John Smith
Legacy Resource Pack but were previously forced to use the English font only.

When I added the horses I was a little lazy, as the skeleton horse was not used I never made a
texture for it, that's now been rectified, I've not done the Zombie horse yet as that's not in the
vanilla game yet but I'm sure that will change.

Fixed a Mojang Cauldron bug that would "cullface" the bottom face, still not fixed in the actual
game though as the bug tracker can be a little slow, please go and upvote the fix.

https://bugs.mojang.com/browse/MC-58405

Also the way Minecraft draws the Cauldron and the Minecraft anti X-Ray block transparency feature,
the Cauldron used to be just the textures listed below displayed for the various sides:-

cauldron_bottom.png
cauldron_inner.png
cauldron_side.png
cauldron_top.png

But now the Cauldron is a model (like the Anvil), which means the legs are now drawn on the inside
too, as the John Smith textures had more slender legs you now get get the white pixels when using
the JSL textures (from the Minecraft anti X-Ray "feature").

The two options are draw the leg textures bigger to suit the default model or make the model file
suit the JSL textures.

I decided to go with the first option as it's a little easier to do but then with some testing found
quite a few issue in the way the textures are drawn on the vanilla Cauldron (stretching and
rotation issues).

After that I then decided to make my own model to better suit the JSL textures (with the slender
legs), with more testing I've also added a custom bottom and inside texture.

Thanks to Alpha_Wolf777 in the forums for the bug report.

I had no plans on testing this pack in Optifine but decided to do some limited testing anyway as
some people are having some issues.

In Minecraft 1.7 onwards, blocks are referred to by name rather than ID number.

Numerical IDs are supported for older blocks currently, but newer blocks will be name only. 
 
Stone blocks for example can be referred to as:-
   1               (not recommended)
   minecraft:stone (full name)
   stone           (name without prefix)
   
The trouble is Optifine still uses ID's so this is why the custom colours are not working correctly.

I've temporarily change the names back to ID's (names are still in the color.properties files but
are commented out) as at least this is currently compatible with both Optifine and MCPatcher but
with new blocks coming along like Granite, these will not even have an ID number and go by just a
name so this will have to change to the newer system in the end anyway.

Some things like the trip wire hooks and lever do not work in Optifine (they will look default).

As the font can display wrong with Optifine, I've removed the two font kerning properties files.

                textures/blocks/glass_pane_top_silver

These two textures are 日本語襖/障子 (Japanese Fusuma/shōji) style sliding panels.

The glass_silver.png texture is based on one of the alternative John Smith Glow
Stone textures, both glass_pane_top_white and glass_pane_top_silver have edges
that match the panes.

The glass_silver.png is 10% transparent and glass_white.png is 5% transparent, this is so it lets
light through for a great nighttime effect, plus the paper panels would let light through in real
life but would not be see through.
